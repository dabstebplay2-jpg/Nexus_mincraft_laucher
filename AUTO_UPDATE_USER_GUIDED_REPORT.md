# Nexus Launcher — User Guided Auto Update

Сделано:
- версия поднята до `0.7.4`;
- `core/updater.py` теперь умеет проверять обновления из двух источников:
  - GitHub Releases API;
  - website `release.json` как fallback;
- если доступны оба источника, выбирается самая новая версия;
- setup installer `.exe` теперь имеет приоритет над portable `.zip`;
- добавлен user-guided auto update:
  - Nexus скачивает Setup EXE в `data/updates`;
  - создаёт `run_nexus_update.cmd`;
  - закрывает лаунчер;
  - запускает установщик после закрытия лаунчера;
- обновлён startup-check:
  - при запуске Nexus сам проверяет обновления;
  - если новая версия есть, пользователь может сразу согласиться на скачивание и запуск установщика;
- обновлена панель `Настройки → Обновления Nexus`:
  - кнопка `Проверить`;
  - кнопка `Скачать`;
  - кнопка `Скачать и обновить`;
  - показ источника: GitHub Releases или сайт release.json;
  - прогресс скачивания;
- если скачан portable ZIP, Nexus не пытается автообновиться и предлагает распаковать вручную;
- обновлён сайт/release.json под версию `0.7.4`.

Как работает:
1. Nexus при запуске проверяет GitHub Releases и website/release.json.
2. Если версия выше текущей, показывает диалог.
3. Пользователь подтверждает обновление.
4. Nexus скачивает Setup EXE.
5. Nexus создаёт update-helper `.cmd`.
6. Nexus закрывается.
7. Helper запускает установщик.
8. Пользователь проходит установку как обычную Windows-программу.

Важно:
- это не тихое фоновое обновление без вопросов;
- обновление запускается только по подтверждению пользователя;
- полностью silent auto-update можно сделать следующим этапом отдельным `NexusUpdater.exe`.
