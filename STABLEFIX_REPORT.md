# Nexus Launcher stable fix

## Зачем этот патч

Этот патч не меняет дизайн. Он убирает места, из-за которых Nexus мог выглядеть «зависшим» или тормозить:

1. Страница «Моды» больше не скачивает иконки перед показом результатов. Раньше при медленном интернете поиск мог ждать десятки HTTP-запросов к картинкам.
2. Лимит первой страницы Modrinth снижен с 24 до 12, чтобы UI отвечал быстрее.
3. Проверка Java больше не выполняется синхронно при старте окна. Теперь Java проверяется при запуске игры или вручную кнопкой в настройках.
4. Добавлены глобальные обработчики ошибок, чтобы скрытые ошибки потоков попадали в лог.
5. Добавлен диагностический запуск: `python main.py --diagnose`.
6. Исправлен вывод версии Java в настройках.
7. Настройки языка/темы используют общий `storage.paths.DATA_DIR`, а не жёсткий путь к папке проекта.

## Главная логика модов

Для `.jar` модов нужна сборка с loader: `fabric`, `forge`, `neoforge` или `quilt`.
Vanilla не загружает файлы из `.minecraft/mods`.

## Как проверить

1. Запусти диагностику:

```powershell
cd C:\Nexus_minecraft_launcher
.\.venv\Scripts\python.exe main.py --diagnose
```

2. Запусти лаунчер:

```powershell
.\.venv\Scripts\python.exe main.py
```

3. Создай новую сборку:

- Minecraft: `1.20.1`
- Loader: `fabric`
- RAM: `4096 MB`

4. На вкладке «Моды» найди `Sodium`, установи его в эту Fabric-сборку.

5. В логах при запуске должно быть примерно:

```text
launch_version=fabric-loader-...-1.20.1
```

Если в логах `launch_version=1.20.1`, значит запущена Vanilla-сборка, и моды не подхватятся.
