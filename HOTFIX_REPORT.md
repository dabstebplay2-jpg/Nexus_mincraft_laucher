# Nexus Launcher hotfix: установка модов

## Что исправлено

1. Запрещена установка `.jar` модов в Vanilla-сборку.
   Раньше лаунчер мог скачать файл в `mods`, но Vanilla Minecraft его не загружал.

2. Убрана опасная логика «скачать последнюю доступную версию мода», если нет совместимой версии.
   Теперь Nexus скачивает только версии, совпадающие с выбранными `minecraft_version` и `loader`.

3. Добавлена проверка совместимости версии Modrinth перед скачиванием:
   - Minecraft version;
   - loader: fabric / forge / neoforge / quilt;
   - тип файла `.jar`/`.litemod` для модов.

4. Исправлена проверка обновлений модов в деталях сборки.
   Был вызов `get_project_versions(..., loaders=..., game_versions=...)`, но метод принимает `minecraft_version` и `loader`.

5. Исправлен callback запуска Minecraft.
   Теперь окно запуска и центр загрузок получают статус/прогресс от `Launcher`.

6. Исправлен `requirements.txt`: файл перекодирован из UTF-16 в UTF-8.

7. Папки данных приведены к единой логике:
   - режим разработки: папка проекта;
   - собранный `.exe`: `%APPDATA%\\NexusLauncher`.

## Главная причина проблемы

По логам видно, что моды скачивались, например Sodium:

`Installed mod file: ...\\.minecraft\\mods\\sodium-fabric-0.5.13+mc1.20.1.jar`

Но в одном из запусков сборка была `loader: vanilla`, поэтому Minecraft запускался как обычный Vanilla и не подхватывал моды из папки `mods`.

## Как проверить после замены файлов

1. Создай новую сборку:
   - Minecraft: `1.20.1`
   - Loader: `fabric`
   - RAM: `4096 MB`

2. Установи мод `Sodium`.

3. Запусти сборку.

4. В логах должно быть:
   - `launch_version=fabric-loader-...-1.20.1`
   - файл мода должен лежать в `.minecraft/mods/` выбранной Fabric-сборки.

## Важно

Если у тебя уже есть старая сборка с названием вроде `test fabric`, но внутри `loader: vanilla`, проще удалить её и создать новую с `Loader = fabric`.
